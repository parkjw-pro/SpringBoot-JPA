# JPA프로그래밍(1)



###### 실무에서는  

- 몇십개의 테이블과 객체가 얽혀서 돌아가게 된다.
- 이것들을 잘 설계하고, 매핑을 정확하게 하는 것이 중요하다.
- JPA를 사용할 때의 장점
- 1. 개발생산성
  2. 유지보수 & 속도



###### 객체를 영구 보관하는 다양한 저장소

RDB, NOSQL, FILE 등등..



###### 객체를 관계형 DB에 저장

객체 -> SQL(개발자) -> RDB



객체의 상속관계와 그나마 유사한건

Table 슈퍼타입, 서브타입 관계



###### 연관관계

객체는 참조를 사용 : member.getTeam()

테이블을 외래 키를 사용: JOIN ON M.TEAM_ID = T.TEAM_ID
